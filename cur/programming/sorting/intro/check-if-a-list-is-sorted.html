You should think about what other good test cases would be. For example, you should think about what should happen if there are no elements or if there is only one element. These test cases check the corner cases. The normal test cases, which most people will remember to make, check whether the block works for a list with a lot of elements. However, it is as important to think about what it will do for no elements or one element. This makes your script more robust: it allows your script to act properly in case of unexpected or tricky input.

Boolean values

When returning true or false, make sure that you use the blocks that Scratch and BYOB have provided especially for this purpose, in the Operators tab:


true and false are known as boolean values. Notice that they have a hexagonal shape, which means they can fit into any hexagonal slots. This also means that any block with a hexagonal shape returns boolean values, and in many programming languages -- Scratch and BYOB included -- boolean values are treated as a special boolean type. Boolean values are named after George Boole, a 19th century mathematician who first proposed using true and false as the basis for logical thinking.
