Saving Your Queries

  • Important note when saving: Do this every time you make a query. The web application can only save one query at a time. This is different from Snap! where multiple different connected blocks can exist at once in your workspace. This means after every exercise, you should save your query as either a view or a file if you want to keep a version of it. Otherwise, to remember what you did you can just take a screenshot. To screenshot on a Mac you can press Command-Shift-4 and drag over the area you want to take an image of. On Windows, you can use the Snipping Tool.
  • Save each query as a view (Optional). Make sure the query result and node result is what you want. Saving a view will essentially make a block that shows the result of the query. However, it will not show what blocks you used to make it. Once you make a view, the block will automatically appear in your data sets palette.

  • Save each query as a file .dbs file. (Required for checkoff) This will automatically download the query in your Downloads folder (if not, it should say which folder it will be located in. See picture below). Make sure you do this step if you want to open the query later on in DBSnap because it doesn't save to an account like Snap! does

  • Clear the workspace after each query. The application will not recognize a query result if there are multiple queries in the workspace.

    • How to import your query back into your workspace?

  • Simply press the import query button and choose the according .dbs file that was saved to your Downloads folder.
