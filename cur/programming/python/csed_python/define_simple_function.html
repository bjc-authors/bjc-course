Defining a function in Python is easy! Compare the Snap! program on the left with its Python equivalent on the right. Both programs take an integer as an argument, and print out all the numbers between 1 and that number.

    Function definition: Like in Snap!, a function is separated into two parts. The first part is the function header, which specifies the name of the function, and any arguments (a.k.a. input variables, parameters) of the function. Notice that in Python, the line must begin with the keyword def and end with a :. This lets the Python interpreter know that the indented lines are a function header. The second part of a function is the function body. Notice that the two bodies look very similar.

    Create a new file in Sublime Text and give it any name you want, e.g. count.py. Inside your file, add the following code to your file: 

    def count_up(num):
    x = 1
    while x <= num:
        print(x)
        x = x + 1

count_up(4)

    Try running the file, and you should see something like the following happen:

2
3

    Before moving on, let's consider how our Python and Snap! programs are different:

    Declaration and setting of variables: In Snap!, we had to create a script variable before setting it. In Python, we can do this all at once by simply setting a variable equal to a value, e.g. x = 1.

    while vs. repeat until: In Python, instead repeat until, we have a block called while. As you know, repeat until goes until the condition is false. while does the opposite thing, going until the condition is true.

    Less than or equal: In Python, we have a special operator called <= which returns true if the left operand is less than or equal to the right operand. For example x <= num returns true as long as x is less than or equal to num, and false if x is greater than num.

    Definitions and calls can go in the same file: When we make a Snap! program, the block definitions don't show up on the main page. However, inside a Python file, function definitions and calls to those functions can go in the same file. For example, in the program above, the definition of count_up is in the same file as the call to this function at the very end that reads count_up(4).

    In the real world, programmers often put function definitions into separate files, but for simplicity, we will not do this today.
