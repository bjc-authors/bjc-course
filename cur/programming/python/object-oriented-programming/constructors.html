Recall the code thus far for our book class: 

    
class Book:
    language = 'English'
    worn_down = False

    In addition to having attributes like above, we can also include functions within our class. We call these functions methods; these methods are specific to our class and can only be used by instances of our class. There is a special method which we use to initialize, or create, objects, known as the __init__ method (two underscores on each side). This method is called a constructor and very useful in initializing objects with instance attributes!

    Here is what our code would look like if we wanted to initialize an object with certain attributes : 

    
class Book:
    language = 'English'
    worn_down = False

    def __init__(self, genre, title, author, publication_year):
        self.genre = genre
        self.title = title
        self.author = author
        self.publication_year = publication_year

    Now, when we create a Book object, we need to make sure every argument is correctly filled, which would be 4 in this case. 

    
>>> hunger_games = Book('Dystopian fiction', 'The Hunger Games', 'Suzanne Collins', 2008)
>>> hunger_games.genre
'Dystopian fiction'
>>> hunger_games.publication_year
2008
>>> hunger_games.language
'English'


    Key things to note:

    1. If you're wondering why we didn't pass anything in self, it's because Python automatically does that for you. How neat!
    2. Inside the constructor, you'll notice that we add self in front of our class attributes. Since our constructor has the same argument names as our instance attributes, we use self to let Python know that we're referring to the object and its attributes, not the argument names. If we didn't add this, Python would simply define the argument as the argument.
    3. We must ensure that any arguments we pass into In our case, we need to make sure we put quotes around our string if we want Python to treat the input as a string.
    4. Even though we never defined our book's language, it's automatically set to English because we defined it outside of the constructor. In general, class attributes don't need to be defined in the constructor, and every instance of a class shares the attribute.
